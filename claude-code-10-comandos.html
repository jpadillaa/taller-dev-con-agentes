/ claude code · guía de bolsillo

Doce comandos para empezar bien

Ordenados según el momento en que un desarrollador los encuentra, desde abrir la terminal por primera vez hasta el cierre de una sesión productiva.

FASE 01Acceso
  1. 01
    $claude
    Abrir la herramienta

    Se escribe en la terminal del sistema, dentro de la carpeta del proyecto, para iniciar la sesión interactiva de Claude Code. Es el único comando que no empieza por barra.

    → Cada vez que se comienza a trabajar.

  2. 02
    /login
    Autenticarse

    Conecta la sesión con tu cuenta de Anthropic. Aparece en el primer arranque y sirve después para cambiar de cuenta cuando hace falta.

    → En el primer uso, o al alternar entre cuentas.

FASE 02Arranque
  1. 03
    /help
    Descubrir el terreno

    Lista todos los comandos disponibles con una breve descripción. Es lo primero que se escribe cuando aún no se sabe qué existe.

    → El día cero, antes de pedir cualquier cosa.

  2. 04
    /init
    Dar memoria al proyecto

    Analiza el repositorio y genera el archivo CLAUDE.md, de modo que Claude entienda la estructura y las convenciones del código desde el inicio.

    → Al abrir un proyecto por primera vez.

  3. 05
    /permissions
    Definir límites de confianza

    Establece qué herramientas y acciones puede ejecutar Claude sin pedir confirmación. Aquí se decide cuánta autonomía concederle.

    → Antes de dejarlo trabajar sin supervisión constante.

FASE 03Trabajo
  1. 06
    /model
    Elegir la herramienta correcta

    Cambia el modelo según la tarea, reservando los más potentes para los problemas complejos y los ligeros para el trabajo rutinario.

    → Cuando el costo o la dificultad de la tarea lo justifica.

  2. 07
    /plan
    Pensar antes de tocar

    Activa el modo de planificación para que Claude proponga una estrategia antes de modificar archivos, lo que evita errores costosos en cambios grandes.

    → Frente a una refactorización o una funcionalidad amplia.

FASE 04Sesión
  1. 08
    /context
    Medir cuánto cabe

    Muestra qué porción de la ventana de contexto está ocupada. Sirve para anticipar el momento en que la sesión empieza a saturarse.

    → Cuando las respuestas comienzan a perder precisión.

  2. 09
    /compact
    Comprimir sin perder el hilo

    Resume el historial conservando lo esencial y libera espacio de contexto. Permite sostener una sesión larga sin reiniciarla.

    → En tareas extensas, cada cierto tiempo de trabajo activo.

  3. 10
    /clear
    Empezar de cero

    Reinicia la conversación a un contexto vacío para abordar una tarea nueva sin arrastrar el ruido de la anterior.

    → Al cambiar por completo de objetivo.

FASE 05Calidad y cierre
  1. 11
    /review
    Control de calidad

    Solicita una revisión estructurada de los cambios, en busca de errores, riesgos de seguridad y problemas de mantenibilidad antes de integrar.

    → Justo antes de confirmar o abrir un pull request.

  2. 12
    /cost
    Cerrar las cuentas

    Muestra el consumo de tokens y el costo de la sesión. En las versiones recientes funciona como alias de /usage.

    → Para entender el gasto y ajustar el flujo de trabajo.

Nota. La disponibilidad y el nombre de cada comando varían según la versión, el plan y la plataforma. Escribe / en la terminal de Claude Code para ver los de tu instalación.
